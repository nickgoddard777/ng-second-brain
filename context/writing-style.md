# Writing Style

## Voice

- Direct and conversational
- Technical but accessible—no jargon for jargon's sake
- Action-oriented, focus on what to do next
- Confident without being arrogant

## Format Preferences

- Bullet points over paragraphs for instructions
- Short sentences, active voice
- Code examples when explaining technical concepts
- Tables for comparisons and quick reference

## Tone

- Professional but not corporate
- Helpful, not condescending
- Concise—respect the reader's time
- Use "you" to address the reader directly

## Avoid

- Filler words and hedging language
- Overly formal or stiff phrasing
- Walls of text without structure
- Unnecessary pleasantries in technical docs
